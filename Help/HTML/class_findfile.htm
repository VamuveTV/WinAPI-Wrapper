WinAPI Wrapper

FindFile

The FindFile class helps finding files using a filename pattern in a specific directory. It extends HANDLE objects - find handles. See Searching for Files for the details.

The FindFile class has the following functions:

Function Related WinAPI function Description
FindFile FindFirstFile Find the first file using the specified pattern.
~FindFile FindClose Calls Close in order to close the find handle.
operator HANDLE - Returns the find handle.
IsOpen - Validates the handle.
FindNext FindNextFile Finds next file.
GetFindData - Returns reference to the internal WIN32_FIND_DATA structure.
Close FindClose Closes the handle.
SetTo FindFirstFile Starts a new search and finds the first file, closing the previous handle.
begin
end		 - These functions return simple iterator objects that can be used in the same way that STL iterators are used to iterate over entries in a directory.
ForEach - This template function calls a specified handle object on every directory entry. See the implementation of GetAll to see how to use it.
GetAll - Returns a list of all filenames or WIN32_FIND_DATA entries in a vector. There are two versions of these functions - member and static.

Examples

Find all "cpp" files:

vector< string > filenames;
FindFile finder( "*.cpp" );
finder.GetAll( &filenames, FindFile::FILES );

The same effect as the above code:

vector< string > filenames;
FindFile::GetAll( "*.cpp", &filenames, FindFile::FILES );

Finds all subdirectories and stores their full entry descriptions:

vector< WIN32_FIND_DATA > dirs;
FindFile::GetAll( "*.*", &dirs, FindFile::DIRECTORIES );

Finds all files in the root directory C:\

vector< string > filenames;
FindFile::GetAll( "C:\\*.*", &filenames, FindFile::FILES );

Displays all subsequent entries (files and directories) in a directory by passing them to an appropriate handler (cmdline application):

class MyHandler {
public:
    void operator()( const WIN32_FIND_DATA& fd ) {
        const bool b_directory = (fd.dwFileAttributes & FILE_ATTRIBUTE_DIRECTORY) != 0;
        cout << (b_directory ? "directory " : "file ") << fd.cFileName << endl;
    }
};

FindFile finder( "*.*" );
finder.ForEach( MyHandler() );

Achieves exactly the same thing, but displays only subdirectories:

FindFile::ForEach( "*.*", MyHandler(), FindFile::DIRECTORIES );

The same effect as above, but not using the template functions:

FindFile finder( "*.*" );
if ( finder.IsOpen() ) do {
    const WIN32_FIND_DATA& fd = finder.GetFindData();
    const bool b_directory = (fd.dwFileAttributes & FILE_ATTRIBUTE_DIRECTORY) != 0;
    cout << (b_directory ? "directory " : "file ") << fd.cFileName << endl;
} while ( finder.FindNext() );

The same effect, but using iterators:

FindFile finder( "*.*" );
for ( FindFile::iterator it=finder.begin(); it != finder.end(); it++ ) {
    const WIN32_FIND_DATA& fd = it;
    const bool b_directory = (fd.dwFileAttributes & FILE_ATTRIBUTE_DIRECTORY) != 0;
    cout << (b_directory ? "directory " : "file ") << fd.cFileName << endl;
}
