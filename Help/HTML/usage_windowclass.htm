WinAPI Wrapper

Using custom window classes

A window class or WNDCLASS in Windows keeps GDI objects for windows that use it.

The WinAPI Wrapper has a default window class for windows derived from the Window class. But you can override this class if you want to specify, for example, your own application icon or window background.

class SomeWindow: public Window {
public:
    SomeWindow() {
        static WindowClass wc( LoadIcon( __MODULE_HANDLE, IDI_MYICON ), 0 );
        Create( "Title", wc );
    }
};

In the above example a WNDCLASS is passed to the Create function while the window is being created. The WNDCLASS is created using an object of WindowClass class. The object's constructor takes in this case an icon loaded from a resource and background brush 0 (no background). WindowClass objects should be created statically in order to avoid them being initialized over and over - such objects has to be initialized only once.

Note that if you create an MDI child window derived from the MDIChildWindow class, you have to use the MDIChildWindowClass class instead.
