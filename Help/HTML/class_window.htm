WinAPI Wrapper

Window

The Window class is a C++ extension to the WindowProc C user procedure. The class is derived from GenericWindow. See the Remarks section for information how to use this class.

The Window class has the following methods:

Function Related WinAPI Message Description
HandleMessage all Corresponds to WindowProc and processes all other messages not processes by the other methods below. In pure wrapper mode this method processes ALL messages.
OnCreate WM_CREATE Called when the window is being created. Typically creates child windows (controls, etc.).
OnClose WM_CLOSE Called when the window is begin requested to close. Calls Window::Destroy() is the user accepts the destruction. Rarely used for main application windows.
OnDestroy WM_DESTROY Called during the destruction of the window. In a main application window typically calls PostQuitMessage.
OnCommand WM_COMMAND Processes commands coming from miscellaneous controls (buttons, menus, toolbars, etc.).
OnNotify WM_NOTIFY Usually called when changes occur in children (controls).
OnPaint WM_PAINT Repaints the dirty regions of window client area, typically using PaintDC.
OnSize WM_SIZE Obtains information while the window is being resized.
OnMouseMove WM_MOUSEMOVE Receives mouse movement notification.
OnMouseDown WM_LBUTTONDOWN WM_RBUTTONDOWN WM_MBUTTONDOWN Called when a mouse button is depressed.
OnMouseUp WM_LBUTTONUP WM_RBUTTONUP WM_MBUTTONUP Called when a mouse button is released.
OnMouseDblClk WM_LBUTTONDBLCLK WM_RBUTTONDBLCLK WM_MBUTTONDBLCLK Called when a mouse button is depressed a short period of time after the first depression.
OnKeyDown WM_KEYDOWN Called when a key on a keyboard is depressed.
OnKeyUp WM_KEYUP Called when a key on a keyboard is released.

Remarks

Using the Window class simplifies the process of implementing window functionality. When using it, the following rules must be taken into account:

  • Messages that would normally be processed by WindowProc are handled by the methods of the Window class.
  • In pure wrapper mode, the OnXXX methods do not exist and all messages go to the HandleMessage method.
  • The object of the Window-derived class must be created dynamically with the new operator. It may not be destroyed using the delete operator.
  • The object of the Window-derived class is always automatically destroyed when the window is destroyed, i.e. after processing the last message, which is WM_NCDESTROY.
  • If you derive a class from Window and you do not intend to derive from it more (i.e. you create a final class), you should call Create in the constructor. However if you still are going to derive from it, don't call Create, because the method in the latter class will not be called at all during the construction.
  • To specify your own window class for the window, you must use the WindowClass class in order for the message dispatch mechanizm to work.
