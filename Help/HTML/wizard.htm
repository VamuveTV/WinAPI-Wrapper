WinAPI Wrapper

AppWizard

If the WinAPI Wrapper is used in Microsoft Visual Studio .NET environment, there is an additional AppWizard for it. The AppWizard creates a simple, skeleton application. It can serve educational purposes as well as for a quickstart project.

The AppWizard appears under the name "WinAPI Wrapper" in the Visual C++ Projects section in the New Project dialog box. The AppWizard lets you select the following options when creating a skeleton application:

  • Application (or main window) type. You can select either SDI (a regular main window derived from the Window class), MDI (MDI application with main window derived from the MDIFrameWindow class and a sample document window derived from the MDIChildWindow class) or Dialog-based application (the main window is a modeless dialog derived from the ModelessDialog class).
  • What message handlers to create. With the regular WinAPI Wrapper settings to handle chosen messages there are methods created in the main window class (or in the document window class in case of an MDI application). If a pure wrapper configuration is used, however, there is only one method HandleMessage created, but chosen messages appear in its switch statement.
  • Additional options. You can choose to create an empty accelerator table and to take it into account in the message loop, you can select a pure wrapper configuration, you can drop support for STL in the WinAPI Wrapper header, you can make an application with precompiled headers (faster compile) or you can choose your application to be skinnable in Windows XP.
  • Extended main window options. You can choose to create custom window class, menu, toolbar on optional rebar or statusbar.
