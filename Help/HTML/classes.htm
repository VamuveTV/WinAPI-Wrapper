WinAPI Wrapper

WinAPI Wrapper classes

The following classes are part of the WinAPI Wrapper.

The classes have several types:

  • helper - A helper class. Making objects of this class makes no sense.
  • autodestr. - An autodestructive class. The destructor of the class destroys the actual Windows object that the class enhances.
  • transparent - A transparent class. The class provides simplified WinAPI functionality for Windows objects but does not destroy them.
  • alloc only - A class that serves the purpose of allocating new objects only. The class does not destroy the objects, because they are indestructible (like window classes).
  • dynamic - A dynamically allocated class. You don't use this class directly, but rather derive your own classes from it. Later on, you dynamically allocate objects of your derived class and you never create them statically (e.g. on the stack).
  • extractable - An autodestructive class with an option of extracting the object from it to avoid its destruction through the destructor.

The class hierarchy is not represented by a diagram because it is not really a real class hierarchy, except maybe for the windowing classes. Some classes are simply derived from NonCopyAble just to prevent copying their objects.

The descriptions of functions are very brief. You can find complex descriptions in the WinAPI documentation, which is referenced by appropriate links aside by each function. The function arguments are not described, because most of the times their naming and meaning is exactly the same as of the related WinAPI functions and you have appropriate tools in your development environment to view the lists or arguments as tooltips or you can quick jump to the definitions in the header file.
