Dialog

The Dialog class is derived from the Window class. It is designed to handle modal dialogs and it extends the DialogProc C procedure with C++ functionality. For modeless dialogs use the ModelessDialog. See the Remarks section for details.

The Dialog class has the following functions and methods:

Function Related WinAPI phenomena Description
Dialog - Initializes the object with an identifier of the dialog resource.
EndDialog EndDialog Destroys the dialog box. Use this for modal dialog boxes instead of GenericWindow::Destroy().
Go DialogBoxParam Executes the dialog box.
OnInitDialog WM_INITDIALOG This method is issued during the creation of the dialog box instead of WM_CREATE.

Remarks

A dialog box never receives WM_CREATE, WM_PAINT, WM_KEYDOWN or WM_KEYUP messages, so the corresponding methods from Window are hidden in the Dialog class. The rest of the methods are the same as in Window. However, unlike a Window-derived object, objects based on Dialog-derived classes can be created statically and are not automatically destroyed - they must be destroyed explicitly.
