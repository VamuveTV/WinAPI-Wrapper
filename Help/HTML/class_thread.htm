Thread

The Thread class simplifies the use of threads. See Multiple Threads for details.

The Thread class has the following functions:

Function Related WinAPI function Description
Thread - Does nothing. Use Create() to actually create the thread.
Create CreateThread Creates thread in either running (default) or suspended state. You can call it in the constructor of a derived (final) class.
~Thread WaitForSingleObject CloseHandle Waits until the thread terminates and then destroys all its remainings.
Run ThreadProc An abstract method for derived classes. It is called as the actual thread procedure.
Suspend SuspendThread Suspends a running thread.
Resume ResumeThread Resumes a suspended thread.
Exit ExitThread Stops execution of a thread as if the thread procedure ended.
Sleep Sleep Suspends thread execution for a specified amount of time.
SetPriority SetThreadPriority Sets thread priority.
GetPriority GetThreadPriority Returns current thread priority.
SetAffinityMask SetThreadAffinityMask Selects processors on which the thread can run.
SetIdealProcessor SetThreadIdealProcessor Selects the preferred processor for the thread.
Terminate TerminateThread Called from another thread breaks execution of the thread and kills it. The use of this function is strongly discouraged.
